# CLAUDE.md — 콘텐츠 자산화 시스템 (fastpoten-content-assets)

> 이 저장소의 모든 클로드 코드 세션이 가장 먼저 읽는 총괄 문서.
> 사용자는 **비개발자**다. 모든 설명·결정 질문은 코드 용어 없이 화면·사용 흐름 기준 일상 언어로 한다. 구현 방법은 사용자에게 묻지 않는다.

## 0. 이 프로젝트가 뭔가
부서가 수년간 네이버 카페에 올린 글(약 11,000건)을 **추출·정제·분류해 "콘텐츠 자산 창고(DB)"로 만드는** 시스템. AI 원고·이미지 "생성"은 다음 단계(생성 단계)이며 이 저장소 범위가 아니다.
- 기존 CRM(fastpoten-CRM)과 **완전 별개 저장소·별개 DB**. CRM 코드는 절대 건드리지 않는다.
- 기획 확정본: `docs/서비스기획서-v9.md`(무엇을) + `docs/개발기획서-v1.md`(어떻게·차수). **두 문서가 단일 기준.**

## 1. 진짜 불변 (깨면 안 됨)
1. **개인정보 마스킹**: `body_publishable_reference`와 모든 "참고용" 산출물에서 전화번호·오픈채팅·카페닉네임·직원 실명·OO쌤/멘토/팀장 호칭을 반드시 가린다. 개인정보 든 이미지는 `image_pattern_only`(원본 재사용 금지).
2. **계정정보 시트 미반입**: 엑셀의 `계정 정보` 탭 내용(비밀번호 등)은 어떤 형태로도 DB·코퍼스에 넣지 않는다. 업로드 계정 라벨(account_id)은 '현황' 시트에서만.
3. **순위·조회수는 "참고 신호"**: 성과·품질 지표로 다루는 코드·문구 금지. 등급제(P1/P2/P3)·length_penalty 금지.
4. **본문은 파일에만**: DB에 본문 텍스트 컬럼을 만들지 않는다(경로만). 원문(body_raw)은 절대 수정하지 않는다.
5. **네이버 자동 추출은 백로그**: 사용자가 명시 요청하기 전에는 Playwright/스크래핑 코드를 만들지 않는다(BACKLOG.md 참조).
6. **간결(YAGNI)**: 새 의존성·추상화 전에 표준/기존 기능 먼저. 단 입력검증·마스킹·데이터 손실 방지는 간결화 대상 아님.
7. DB 마이그레이션은 멱등(ensure 패턴). 컬럼 삭제 금지(숨김 OK).

## 2. 현재 결정 (재검토 가능 — 변경은 사용자만)
- 파일럿은 **수동 투입(inbox/)** 방식: 사람이 글 텍스트+이미지를 넣으면 시스템이 표준양식으로 정리.
- 검토 화면은 요약(A안), 저장은 전체(B안, v9 부록 A 형식).
- 임베딩은 외부 AI API + numpy(전용 벡터DB 없음). 구체 API는 CA-5 착수 시 사용자와 1회 결정.

## 3. 작업 방식
- 차수 단위 진행: `docs/개발기획서-v1.md` §5의 **CA-1부터 순서대로**. 한 번에 한 차수.
- 에이전트 사이클: planner(차수 확인·분해) → implementer(구현) → reviewer(회귀·과잉 검토). 충돌 시 사용자에게 선택지 올림.
- 각 차수 완료 시: 자체 테스트 통과 + 작업노트 MD(`docs/notes/`) + 사용자 눈검수 안내(개발기획서 §6의 해당 항목을 일상 언어로).
- `data/`(DB)와 `corpus/`(글·이미지)는 git에 올리지 않는다(.gitignore).
