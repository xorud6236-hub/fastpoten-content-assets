# 화면 명세 — 품질 확인 뷰어 "분석" 메뉴(참고 신호 대시보드) + 목록 컬럼 추가

작성: 2026-07-13 / 화면설계(designer). **구현 아님, 화면 명세만.**
대상 파일: `src/viewer.py` (새 화면 `render_analysis` 추가, `render_list` 컬럼 2개 추가).
이어받는 문서: `docs/plans/20260707-cafe-extractor-web-viewer-design-v2.md`(씨앗 패턴·목록·접근성) · `docs/plans/20260709-구조DNA-샘플링-방향.md`(분석 배경·발견).

## 왜 만드나
사용자(비개발 관리자 1인)가 추출된 우리 카페 글의 **조회수(참고 신호)** 로 "어떤 주제·누가 잘 되나"와 "형식이 조회수와 관계 있나"를 **한 화면에서** 보고 싶어 한다. 지금 뷰어엔 글 1건씩 보는 화면만 있고, 여러 글을 가로질러 보는 화면이 없다.

## 이 화면이 반드시 지키는 것 (불변 재확인) ★
- **불변 3 — 조회수는 "참고 신호"**: 화면 어디서도 조회수를 "성과"로 부르지 않는다. 조회수·하루당 조회수·키워드/담당자 묶음 수치 모두 **참고 신호**다. 섹션 제목·안내에 "성과"라는 말을 쓰지 않는다(→ 결정 1에서 제목 확정).
- **순위(rank)는 화면에 없다**: 카페 순위값이 실제 조회수와 거의 맞지 않으므로(근거: 구조DNA 노트, r≈-0.2) 정렬 기준·표 컬럼 어디에도 rank를 쓰지 않는다.
- **읽기 전용**: 입력칸·수정·발행 버튼 없음. 정렬/필터는 링크(주소 바뀜)로만.
- **개인정보 마스킹 유지 · 담당자 실명 표시 허용**: 이 화면은 글 본문을 통째로 띄우지 않는다(제목·집계 숫자·담당자 이름만). 담당자 실명은 내부 검수용으로 표시 허용(기존 결정). 행을 눌러 들어가는 상세 화면의 마스킹 규칙은 v2 그대로.
- **색은 tokens.css 변수만**. 새 색 토큰 추가 없음.

## 용어 사전 (word-drift 방지) ★ — 화면 전체·향후 화면 공통
| 화면에 쓰는 말 | 뜻 | 쓰면 안 되는 말 |
|---|---|---|
| **조회수 (참고 신호)** | 카페에서 뜯은 조회수 | 성과, 성과 등급, PV, P1/P2/P3 |
| **하루당 조회수** | 조회수 ÷ 게시 경과일 (오래된 글 보정용) | 일당조회, per-day, 일당 |
| **키워드** | 글의 주제어(추출된 keyword) | 태그, 토픽 |
| **담당자** | staff_name(실명) | 작성자, 필자 |
| **형식 지표** | 문단 수·이미지 수·글자 수 묶음 | 구조 DNA, 파싱 결과 |
| **참고 신호 분석** | 이 화면 이름 | 성과 대시보드, 애널리틱스 |

코드 용어(정제/파싱/렌더링/상관계수 등)·등급 용어는 화면 어디에도 쓰지 않는다. (상관값 r은 섹션 4에서만, "거의 관계 없음" 같은 일상어와 **함께** 괄호로만 노출 — 결정 4 참고.)

## 네비게이션 (목록 ↔ 분석 ↔ 상세)
지금 상단 띠엔 화면 이동 메뉴가 없다(목록은 제목만, 상세는 `← 목록으로`만). **작은 메뉴 2칸을 상단 띠에 추가**한다.
- 목록 화면·분석 화면의 상단 띠 왼쪽에: **`글 목록` · `분석`** 두 글자 링크. 지금 보는 화면은 굵게/밑줄로 "여기 있음" 표시(색만으로 구분하지 않음 — 글자 상태로도).
- 분석 화면 주소: `/analysis`. 목록은 지금처럼 `/`.
- 분석 표의 **각 글 행을 누르면 기존 상세(좌우 대조) 화면 `/post?id=…`** 으로 간다. (새 상세 화면 안 만듦 — 기존 재사용.)
- 상세 화면 상단의 `← 목록으로`는 그대로. (분석에서 들어가도 목록으로 나가면 됨 — 뒤로가기도 브라우저 기본 동작으로 됨. 별도 "분석으로" 링크는 이번에 안 만든다: 간결 사다리 3단, 브라우저 뒤로가기로 충분.)

가장 흔한 동작 = "분석 표에서 잘 된 글을 눌러 실물 확인". 그래서 표의 **행 전체가 클릭 영역**(기존 `.listrow` 링크 패턴 그대로)이라 손이 가장 적게 닿는다.

## 배치 — 분석 화면 (위 → 아래)

1. **상단 고정 띠(스크롤해도 붙음)** — 왼쪽 메뉴 `글 목록 · 분석`(분석 강조) · 가운데 제목 `참고 신호 분석` · 오른쪽 배지 `분석 대상 NNN건`(조회수가 있는 추출 완료 글 수, 예: 699건).

2. **안내 두 줄(가로 전체)** — 이 화면이 무엇이고 무엇을 조심해서 봐야 하는지.
   - 1줄: `추출된 우리 카페 글의 조회수(참고 신호)로 "어떤 주제·누가 많이 읽혔나"를 봅니다.`
   - 2줄(작은 회색): `조회수는 성과가 아니라 참고 신호입니다. 카페 순위는 실제 조회수와 거의 맞지 않아 이 화면에서 뺐습니다.`

3. **조절 바(가로 전체)** — 기존 `.filters` 링크 패턴 재사용(주소 바뀜, JS 없음).
   - 정렬: `조회수 순` · `하루당 조회수 순` (지금 고른 것 굵게). — 섹션 1 표에 적용.
   - 기간: `전체` · `올린 지 30일 지난 글만` (지금 고른 것 굵게). — 전 섹션 집계에 적용(최근글 편향 보정). (넣을지 = 결정 3.)
   - 조절 바 아래 작은 회색 주의: `하루당 조회수는 최근에 올린 글이 높게 나오는 경향이 있어요(조회가 초반에 몰림). 오래된 글과 견줄 땐 '조회수 순'도 함께 보세요.`

4. **섹션 1 — 글별 조회수 (참고 신호)** [표, 행 클릭 → 상세]
   - 제목: `조회수 높은 글` + 옆 작은 회색 `정렬: 조회수 순 / 하루당 조회수 순 (위 조절 바)`.
   - 표 컬럼(왼→오): **제목 · 키워드 · 담당자 · 조회수 · 하루당 조회수 · 형식 · 작성일**
     - `제목`: 굵게·주색(누르면 그 글 상세로). 길면 한 줄 말줄임.
     - `형식`: `3문단·4장`처럼 한 칸에 합쳐 표기(칸 수 절약). 이미지 없으면 `3문단·0장`.
     - 숫자 칸(조회수·하루당)은 오른쪽 정렬, 천단위 쉼표(예: 5,976). 조회수 없으면 `-`.
     - `하루당 조회수`는 소수 1자리(예: 48.8).
   - 몇 줄 보일지 = 결정 5(상위 N vs 전체).
   - 기존 `.listhead/.listrow` 그리드 패턴 재사용하되 컬럼 구성이 달라 **분석 전용 그리드 비율**을 둔다(목록 표와 다른 표이므로 변형 클래스 허용 — drift 아님).

5. **섹션 2 — 키워드별 조회수** [묶음 집계 표, 행 클릭 안 됨]
   - 제목: `키워드별 조회수` + 작은 회색 `글이 2건 이상인 키워드만 · 평균 조회수 높은 순 상위 N`.
   - 컬럼: **키워드 · 글 수 · 평균 조회수 · 합계 조회수 · 평균 하루당 조회수**
   - "어떤 주제가 많이 읽혔나"를 본다. 정렬 = 평균 조회수 높은 순(고정). (평균 vs 합계 어느 걸 위로 = 아래 참고: 합계는 많이 쓴 키워드가 유리하므로 **평균**을 기준으로 정렬.)
   - 행은 누를 수 없다(키워드엔 글이 여럿이라 한 글로 못 감). 키워드 눌러 그 글만 목록으로 거르기 = 결정 6.

6. **섹션 3 — 담당자별 조회수** [묶음 집계 표, 행 클릭 안 됨]
   - 제목: `담당자별 조회수` + 작은 회색 `평균 조회수 높은 순`.
   - 컬럼: **담당자 · 글 수 · 평균 조회수** (요청 명세대로 최소).
   - 담당자 29명이라 전부 표시(스크롤 짧음). 실명 표시(내부 검수 허용).

7. **섹션 4 — 형식과 조회수의 관계** [정직 섹션]
   - 제목: `형식과 조회수, 관계가 있을까?`
   - (a) 관계 세 줄 — 각 줄: `문단 수 ↔ 조회수 : 거의 관계 없음 (r=-0.08)` / `이미지 수 ↔ 조회수 : 거의 관계 없음 (r=-0.08)` / `글자 수 ↔ 조회수 : 거의 관계 없음 (r=+0.04)`. (r 값은 실제 계산값을 넣음.)
   - (b) 비교 표 — 컬럼: **구분 · 평균 문단 · 평균 이미지 · 평균 글자수**, 행 2줄: `조회수 상위 25% 글` / `조회수 하위 25% 글`. 두 줄이 거의 같게 나옴(예: 문단 2.8/2.9 · 이미지 3.8/3.8).
   - (c) **정직 문구(강조 박스)**: `글의 형식(문단 수·이미지 수·글자 수)만으로는 많이 본 글과 아닌 글을 가를 수 없습니다. 조회수를 가르는 건 형식보다 "무엇을 다뤘나(주제·시의성)"에 더 가깝습니다.`
   - r 값을 어떻게 보여줄지(숫자 노출 여부) = 결정 4.

## 상태 4종 (분석 화면)
- **비었을 때(첫 사용/데이터 없음)** — 조회수 있는 추출 완료 글이 0건: 섹션들 대신 안내 박스(`.state`):
  `아직 분석할 글이 없습니다. 글을 추출해서 조회수를 확보한 뒤 이 화면을 새로고침하세요.`
  상단 배지는 `분석 대상 0건`.
- **일부만 있을 때**: 있는 글(예: 699건)만으로 집계하고, 상단 배지에 실제 건수를 그대로 보인다(숨기지 않음). 섹션 2에서 2건+ 키워드가 하나도 없으면 그 섹션만 `아직 여러 번 쓴 키워드가 없어 키워드별 비교를 만들 수 없습니다.`
- **불러오는 중**: 서버가 바로 그려 대개 안 보임. 집계가 잠깐 걸리면 `불러오는 중…` 한 줄(v2와 동일 처리).
- **실패했을 때(자산창고 못 엶)**: 목록 화면과 **똑같은 문구** 재사용 —
  `분석을 불러오지 못했습니다. 자산창고 파일을 찾을 수 없어요. 자산창고를 먼저 만든 뒤 다시 열어주세요.` (코드 용어 노출 금지.)
- **성공**: 조절 바 + 섹션 1~4 전부 표시.

## 문구 (실제 문장 — 그대로 사용)
- 화면 제목: `참고 신호 분석`
- 상단 배지: `분석 대상 {n}건`
- 안내 1줄: `추출된 우리 카페 글의 조회수(참고 신호)로 "어떤 주제·누가 많이 읽혔나"를 봅니다.`
- 안내 2줄: `조회수는 성과가 아니라 참고 신호입니다. 카페 순위는 실제 조회수와 거의 맞지 않아 이 화면에서 뺐습니다.`
- 정렬 라벨: `조회수 순` / `하루당 조회수 순`
- 기간 라벨: `전체` / `올린 지 30일 지난 글만`
- 하루당 주의: `하루당 조회수는 최근에 올린 글이 높게 나오는 경향이 있어요(조회가 초반에 몰림). 오래된 글과 견줄 땐 '조회수 순'도 함께 보세요.`
- 섹션 제목: `조회수 높은 글` / `키워드별 조회수` / `담당자별 조회수` / `형식과 조회수, 관계가 있을까?`
- 섹션 2 부제: `글이 2건 이상인 키워드만 · 평균 조회수 높은 순 상위 {N}`
- 섹션 4 관계 라벨 규칙: `|r|<0.1 → 거의 관계 없음`, `0.1~0.3 → 약한 관계`, `0.3~0.5 → 어느 정도 관계`, `0.5+ → 뚜렷한 관계` (지금은 대부분 "거의 관계 없음").
- 섹션 4 정직 문구: `글의 형식(문단 수·이미지 수·글자 수)만으로는 많이 본 글과 아닌 글을 가를 수 없습니다. 조회수를 가르는 건 형식보다 "무엇을 다뤘나(주제·시의성)"에 더 가깝습니다.`
- 데이터 없음: `아직 분석할 글이 없습니다. 글을 추출해서 조회수를 확보한 뒤 이 화면을 새로고침하세요.`
- 키워드 없음: `아직 여러 번 쓴 키워드가 없어 키워드별 비교를 만들 수 없습니다.`

## 화면 폭 대응 (데스크탑 우선)
- 로컬 PC 1인 도구라 데스크탑 우선(v2와 동일 원칙). 휴대폰 대응은 이번 범위 아님.
- **표가 좁은 화면에서 가로로 삐져나가지 않게**: 각 표는 최소 폭을 두고, 좁아지면 **표만 좌우 스크롤**(표를 감싼 칸에 가로 스크롤). 페이지 전체가 옆으로 밀리지 않게 한다. (표를 세로로 쪼개지 않음 — 숫자 비교 표라 열이 붙어 있어야 읽힘.)
- 긴 제목·긴 키워드는 한 칸 안에서 말줄임/줄바꿈. 상단 띠는 스크롤해도 고정(기존).

## 접근성 기본 (단순화 금지)
- 정렬/기간 "지금 고른 것" 표시는 **색만이 아니라 글자 굵기·밑줄로도** 구분.
- 표의 클릭 대상은 행 전체(넉넉한 세로 높이) — 작은 글자 단독 링크 금지(기존 `.listrow` 그대로).
- 조회수 없음은 색 아닌 `-` 글자로 표기(색만으로 "없음"을 뜻하지 않게).
- 숫자 대비·글자 대비 유지. 읽기 전용이라 입력칸 없음.

## 목록 화면 컬럼 추가 (화면 B 소폭 변경)
현재 목록 컬럼: `제목 · 카페 · 상태 · 가림 · 문단 · 이미지 · 작성일`(7칸).
**두 칸 추가 → `제목 · 카페 · 담당자 · 상태 · 가림 · 문단 · 이미지 · 조회수 · 작성일`(9칸).**
- `담당자`: 카페 다음에. 값은 staff_name(실명, 없으면 `-`). 헤더 문구 `담당자`.
- `조회수`: 이미지 다음, 작성일 앞(숫자 칸끼리 모음). 오른쪽 정렬·천단위 쉼표. 없으면 `-`. 헤더 문구 `조회수`.
- **조회수 = 참고 신호 표기**: 헤더 칸이 좁아 "(참고 신호)"를 다 못 넣으니, 표 위(필터 줄 옆)에 작은 회색 한 줄: `조회수는 참고 신호입니다.` 를 둔다.
- 9칸이 되면서 그리드 비율을 다시 잡아야 함(제목은 여전히 가장 넓게). 좁은 화면 처리는 위 "화면 폭 대응"과 동일(표 가로 스크롤).
- 목록에서 조회수로 정렬할 수 있게 할지 = 결정 2.

## 재사용 목록 / 새로 만드는 것 (간결 사다리)
- **재사용(1~2단)**: tokens.css 색 변수 전부, 상단 고정 띠·배지·`.filters` 링크 패턴(정렬/기간 링크), `.listhead/.listrow`(표+행 클릭), `.panel`+`h2.sec`(섹션 묶음), `.masklist`(집계 줄 스타일 대안), `.state`(빈/실패 안내 박스), `.note-bg/--note-ink`(정직 강조 박스 색).
- **새로(4단, 근거 남김)**:
  - **분석 전용 표 그리드 2종**(섹션1 7컬럼 / 섹션2·3 집계 컬럼) — 근거: 목록 표(9컬럼)와 컬럼 구성이 달라 같은 비율을 못 쓴다. 단 `.listhead/.listrow` **클래스·색·행 높이·hover는 그대로 쓰고 그리드 비율만** 변형(새 색·새 상호작용 없음).
  - **상단 띠 메뉴 링크 2칸** — 근거: 화면 이동 메뉴가 없어 목록↔분석 왕래가 안 된다. 기존 topbar·링크 색 재사용, 새 요소는 "현재 화면 강조" 글자 스타일뿐.
  - **정직 강조 박스**(섹션 4 c) — `--note-bg/--note-ink`(가림 하이라이트에 쓰던 노랑 계열) 재사용, 새 색 없음.
  - **표 가로 스크롤 감싸개** — 새 상호작용 아니라 CSS overflow-x만.
- **새 색 토큰 추가 없음.** 전부 tokens.css 기존 변수 안에서 해결.

## 반복 수정 감지
분석 화면은 신규(첫 등장)라 반복 수정 이력 없음. 다만 **"조회수=참고 신호, 성과 아님" 표기**는 상세·목록·분석 세 화면에 반복되므로, 이미 CLAUDE.md 불변 3으로 규칙화돼 있음(추가 승격 불요). 향후 자사 채널(워드프레스 등) 실측 지표 화면이 생기면 그때는 "성과"로 부를 수 있으니(불변 3), 그 화면과 이 카페 참고 신호 화면의 **말 구분(성과 vs 참고 신호)**을 designer가 재점검할 것.

# 결정 필요 (designer가 정하지 않음 — 사용자 승인 후 implementer 진행)

### 결정 1 — 섹션 제목에서 "성과" 대신 "조회수"로 (불변 3 준수 확인)
요청서 섹션 이름은 "키워드별 성과·담당자별 성과"였지만, 불변 3은 카페 조회수를 성과로 단정 금지한다. 그래서 명세에선 **"키워드별 조회수·담당자별 조회수"**로 바꿔 뒀다.
- (권장) 그대로 "조회수"로 표기 — 불변 3에 맞고, 나중에 자사 채널 "성과" 화면과 헷갈리지 않음.
- (대안) "성과"를 쓰되 매번 "(참고 신호)"를 붙임 — 요청 표현에 가깝지만 말이 길고 불변 3과 아슬아슬.
- **화면 영향**: 섹션 제목이 "조회수"냐 "성과(참고 신호)"냐. (이건 규칙 준수 문제라 권장안으로 기울지만, 부르는 말이라 사용자 확인.)

### 결정 2 — 목록 화면에서 조회수로 정렬할 수 있게 할까
- (권장) 넣는다 — 목록 필터 줄에 `조회수 높은 순` 링크 하나 추가(기존 `?view=` 패턴과 같은 방식). 목록에서도 많이 읽힌 글을 바로 위로.
- (대안) 안 넣는다 — 목록은 지금처럼 최신순 고정, 정렬은 분석 화면에서만. 목록을 단순하게 유지.
- **화면 영향**: 목록 상단에 정렬 링크가 하나 느느냐.

### 결정 3 — "올린 지 30일 지난 글만" 필터를 이번에 넣을까
- (권장) 넣는다 — 하루당 조회수의 최근글 편향을 사용자가 직접 걸러 볼 수 있음(조절 바 토글 1개). 발견된 편향이 실재하므로 유용.
- (대안) 나중에 — 이번엔 전체만. 화면이 더 단순.
- **화면 영향**: 조절 바에 기간 토글이 있느냐 없느냐.

### 결정 4 — 섹션 4에서 상관값 숫자(r)를 화면에 보일까
- (권장) 일상어 + 괄호 숫자 병기: `거의 관계 없음 (r=-0.08)`. 비개발자에겐 말이 먼저, 숫자는 근거로. "왜 관계 없다는데?"의 근거를 남김.
- (대안) 숫자 빼고 말만: `거의 관계 없음`. 더 깔끔하나 근거가 안 보임.
- **화면 영향**: 관계 세 줄에 숫자가 붙느냐. (비교 표는 어느 안이든 유지.)

### 결정 5 — 섹션 1 표에 몇 건을 보일까
- (권장) 상위 N건(예: 50건) + 맨 아래 작은 회색 `상위 50건만 보입니다 (전체 {n}건).` — 699건을 한 표에 다 그리면 화면이 매우 길어짐.
- (대안 A) 전체 다 그리고 표만 세로 스크롤.
- (대안 B) 상위 N + `더 보기` 링크(페이지 넘김). 새 상호작용을 늘려 간결 사다리에 걸림(비권장).
- **화면 영향**: 스크롤 길이와 "전체를 한눈에 vs 상위만". (N 값도 함께 정해 주세요 — 30/50/100 중.)

### 결정 6 — 키워드/담당자 줄을 눌러 "그 글만 목록으로" 걸러 볼 수 있게 할까
- (권장, 이번엔 보류) 누르면 안 됨(집계 숫자만). 목록에 아직 키워드/담당자 필터가 없어 새로 만들어야 함 → 별도 차수.
- (대안) 키워드/담당자 줄 클릭 → 목록이 그 키워드/담당자 글만 보이게 필터(`/?keyword=…` 같은). 탐색이 편해지나 목록 필터 기능을 새로 만들어야 함(범위 커짐).
- **화면 영향**: 집계에서 실제 글로 파고들 수 있느냐(이번 vs 다음 차수).
