## 0. 먼저 — 이 일이 정말 필요한가(YAGNI) + 방향 교정

사용자가 원한 것은 "시기별 트렌드 · 조회수 키워드 · 조회수 높은 글의 키워드 분포 · 글 패턴(DNA)"을
종합해 **최적 콘텐츠를 만들기**다. 그런데 **이미 데이터가 답을 일부 줬다**:

- **형식(문단·이미지·글자수) ↔ 조회수 상관 ≈ 0** (2026-07-13 실측). → "좋은 구조로 뽑기"는 헛다리.
  잘 되는 글은 **형식이 아니라 주제·시의성**으로 갈린다.
- **순위(rank)는 신뢰 불가** (rank↔조회수 r≈-0.2). → 선택 근거에서 제외 확정.
- **우리 무기 = 팩트 정확성**(룰북), 그런데 **룰북 팩트 시트가 아직 비어 있음**(refocus 급소 1).
  팩트가 안 채워지면 생성 원고는 [검수:] 투성이 = 범용 AI와 차이 없음.

→ 그래서 이 로드맵의 **뼈대**는 "구조 DNA로 잘 뽑기"가 아니라
**(1) 주제를 정규화해 "무엇이 언제·얼마나 읽혔나"를 보고 → (2) 룰북 팩트로 정확한 원고를 뽑는다**로 잡는다.

**가장 싸고 큰 첫 조각**: 시기별 트렌드·주제별 조회수는 **추출 없이 전체 1만 건 메타(엑셀)만으로**
상당 부분 나온다. 추출(느리고 12GB·네이버 접근)은 나중에, 필요한 만큼만.

## 3. 키워드 정규화/주제 그룹핑 방안 (D1·D2)

**문제**: keyword 1,829종, 변형 극심(정확일치 그룹핑 무의미).
**방안 — 3층, 지어내지 않고 실데이터 라벨만 사용**:

1. **자동 정규화**(규칙): 공백·대소문자 정리 + 흔한 수식어 꼬리 분리(취업·비용·자격증·자격증취득방법·시험·난이도·전망 등을 "의도어"로 떼어냄) → 핵심어(core) 추출. 예: "사회복지사2급 취업"·"~비용" → core "사회복지사2급".
2. **자동 그룹 매핑**: core를 **룰북 카테고리(`rulebook_categories`, 15종·keyword_examples 보유)에 부분일치로 연결**. 새 라벨을 만들지 않고 이미 있는 카테고리 라벨만 씀.
3. **사람 검수**(D2): 1·2에서 못 묶인 것만 목록으로 → 사용자가 클릭으로 주제 지정. 저장은 매핑 테이블.

- 왜 규칙 먼저인가: 룰북 카테고리·수식어 목록이 **이미 있어** 새 의존성·AI 비용 없이 대부분 잡힌다.
  규칙으로 못 잡는 잔여가 많을 때만 AI 군집(임베딩, CA-5)로 승격 — 그 전엔 안 만든다.
- **데이터 구조 변경 주의**: 매핑 저장 위치는 결정 사항(아래 차수 A). `keywords.category` 재사용 vs 새 매핑 테이블.

## 4. 분석 차원별 산출물 (무엇을 어떻게 계산·표시)

기존 `/analysis` 화면(조회수·키워드별·담당자별·형식상관)에 **주제 정규화를 입히고 트렌드 섹션을 더한다.**

1. **시기별 주제 트렌드** — 정규화 주제 × 월/분기별 **우리 발행 건수** 추이(전체 메타). 표/막대.
   - 라벨: "우리가 언제 무엇을 올렸나(검색 수요 아님)". 추출분 있으면 그 주제의 조회수 추이 보조 표시.
2. **주제별 조회수** — 현 섹션2(키워드별)를 **정규화 주제 단위**로 교체. "사회복지사2급" 흩어진 30건이 한 줄로.
3. **조회수 높은 글의 주제 분포** — 상위 조회수 글들이 어느 주제에 몰리나(주제별 상위글 비중).
4. **글 패턴(DNA)** — 형식 자동집계는 이미 "관계 없음"으로 정직 표기 중(유지). 미세 패턴은 5번 AI 정독으로.

모두 읽기 전용·"참고 신호" 용어(불변 3, viewer-analysis-menu-design 용어사전 준수).

## 5. 글 패턴/DNA 추출 방법 (형식 상관 0을 인정한 뒤)

형식 지표는 버리지 않되(정직 섹션 유지) **판별력이 없음을 확인했으니**, 패턴은 **AI 정독**으로 뽑는다.

- **방법**: 기간대비 조회수 상위 30~50건을 **Claude Code(Max)로 이 자리에서 정독**(추가비용 0, 구조DNA 노트 A단계).
- **뽑는 것(실제 글에서만, 지어내지 않음)**: 제목 훅 유형 · 도입 방식 · 문단 역할 흐름(도입→문제→조건→CTA 등, `role` 필드 이미 있음) · CTA 위치/어투 · **주제·시의성 요소**(시험일정/제도변경/트렌드직업 등 — 상위글 다수가 시의성).
- **산출**: "우리 잘된 글 공통 패턴" 요약 1장 + **원고 예시 후보**(`review_pairs` 테이블 활용 가능).
- 전체 태깅(Haiku/Sonnet Batch, 공준모 ~$7~33)은 **패턴이 검증된 뒤에만** — 지금은 안 함.

## 6. "최적 콘텐츠 생성" 파이프라인 (분석 → 주제선정 → 팩트 원고)

```
[분석] 시기 트렌드(공백 주제) + 주제별 조회수(잘 읽힌 주제)
   ↓ 근거로 주제 선정 (시의성 있는 것 우선 — 상위글 특성)
[무기] 룰북 팩트 확정 ← D5. 팩트 없으면 [검수:] 강제(불변 6)
   ↓
[생성] 원고 초안 (룰북 팩트 정확 · 금지어 차단 · 개인정보 마스킹 · 패턴 예시 참고)
   ↓
[사람] 승인 → 발행은 손으로(불변 7·8, 워드프레스 자동발행은 보류)
```

- field-critic 발견("무기=팩트 정확성")과 직결: **생성의 차별점은 구조가 아니라 정확한 팩트.**
  그래서 **D5(룰북 팩트 적재)가 생성의 전제조건**. 이게 비면 생성 차수는 만들지 않는다.
- 지어내기 금지: 주제·근거는 분석 실데이터에서, 수치는 룰북에서만.

## 7. 차수 분해 (의존 순서대로)

> 표기: 영향범위 / 위험도 / 완료조건. 각 차수는 독립 검증 가능.

### 1차 — 키워드 정규화 (규칙 기반) + 검수 목록  ★ 뼈대 시작, 추출 불필요
- 영향범위: **데이터 구조 변경**(주제 매핑 저장) — 신중. 위험도: 중.
- 영향 파일: 새 매핑 로직 + 저장 테이블(멱등 ensure), (검수 화면은 viewer에 소폭) — *구현 시 확정*.
- 완료조건: **1,829 키워드가 주제 그룹으로 자동 매핑되고, 안 묶인 키워드 목록이 화면/파일로 나온다.**
- 선행 결정: D1·D2.

### 2차 — 시기별 주제 트렌드 (전체 메타)  ★ 가장 싼 값, 추출 불필요
- 영향범위: 되돌리기 쉬움(로컬 파일 viewer.py). 위험도: 낮.
- 영향 파일: `src/viewer.py`(분석 화면에 트렌드 섹션 1개).
- 완료조건: **/analysis에서 주제별·월(또는 분기)별 우리 발행 건수 추이가 "검색 수요 아님" 표기와 함께 보인다.**
- 선행: 1차, D4.

### 3차 — 정규화를 기존 조회수 집계에 적용
- 영향범위: 되돌리기 쉬움(로컬). 위험도: 낮.
- 영향 파일: `src/viewer.py`(섹션2를 주제 그룹 단위로 + 조회수 높은 글의 주제 분포 섹션).
- 완료조건: **/analysis 키워드별 표가 주제 그룹 단위로 집계되고, 상위 조회수 글의 주제 분포가 보인다.**
- 선행: 1차.

### 4차 — 추출 확대 (주제별 표본)  ※ 단독 분리 — 되돌리기 어려움
- 영향범위: **되돌리기 어려움**(네이버 페이지 접근·수 시간·수 GB). 위험도: 중. **네이버 접근 = 사용자 승인 필수(불변 5).**
- 영향 파일: 기존 추출기 재실행(새 스크래핑 코드 아님) + data/·corpus/(git 제외).
- 완료조건: **선정 주제별 상위 N건이 추가 추출되어 조회수·본문이 확보된다(목표 건수는 D3에서 확정).**
- 선행: 1·3차(어느 주제를 표본할지 알아야), D3.

### 5차 — 잘된 글 패턴 정독(AI, Max)
- 영향범위: 되돌리기 쉬움(산출은 문서/라벨). 위험도: 낮.
- 영향 파일: 요약 문서(docs/) + (선택) `review_pairs` 예시 저장.
- 완료조건: **상위 30~50건 정독으로 공통 패턴 요약 1장 + 원고 예시 후보가 나온다.**
- 선행: 3·4차(잘된 글 확정), D6.

### 6차 — 룰북 팩트 시트 적재 (무기 해금)  ※ 데이터 적재, 생성 전제
- 영향범위: **데이터 구조 변경/적재**(멱등). 위험도: 중. 병렬 가능(1~5차와 독립).
- 영향 파일: 새 적재 로직 + 룰북 팩트 테이블(엑셀 ②팩트 시트 → DB) — *구현 시 확정*.
- 완료조건: **주제별 제도 팩트(학점·응시자격 등)가 DB에 들어가, 생성 시 [검수:] 없이 채워지는 비율이 확인된다.**
- 선행: D5. (분석 트랙과 병렬 착수 가능 — 급소라 앞당김 권장.)

### 7차 — 최적 콘텐츠 생성 (주제선정 근거 + 팩트 원고 초안)
- 영향범위: 되돌리기 쉬움(초안 출력, 발행 안 함). 위험도: 중(원고 품질·팩트 정확).
- 영향 파일: 생성 로직(초안 출력) + 마스킹·금지어·룰북 재사용.
- 완료조건: **분석 근거로 고른 주제의 원고 초안이 팩트 정확·금지어 0·개인정보 마스킹으로 나오고, 발행은 사람이 한다.**
- 선행: **1·3·5·6차 전부**(특히 6차 없으면 착수 안 함).

**의존 순서**: 1 → 2·3 → 4 → 5 → 7,  그리고 6은 병렬(생성 전 완료). 
**추출·생성 전 재개 조건**(refocus kill/go): 팩트로 [검수:] 없이 채워지는 비율이 낮으면 7차 차별점 없음 → 재검토.
