# 화면 명세 — 로컬 품질 확인 뷰어 (그린필드 씨앗 패턴)

작성: 2026-07-07 / 화면설계(designer). **구현 아님, 화면 명세만.**
대상 차수: `docs/plans/20260707-cafe-extractor-web-viewer.md` 2차 — 로컬 웹 뷰어(읽기 전용, 품질 확인용).

## 이 문서가 정하는 것 (한눈에)
- 이 프로젝트의 **첫 웹 화면**이다. 참조할 기존 웹 화면이 없어 **그린필드 모드**로 씨앗 패턴(글자 크기·주 색상·간격·배지 1종)을 정의한다. **이후 모든 웹 화면은 여기서 정한 기본형을 따른다.**
- 화면 2개: **(A) 글 1건 상세**(걸어다니는 뼈대 — 먼저 만든다) → **(B) 목록**.
- 성격: **읽기 전용**. 수정·발행·재추출 버튼 없음(불변 8과 얽혀 이번 범위 밖).
- 환경: **관리자 1인이 회사/집 PC에서 로컬로 여는 도구 → 데스크탑 우선.** 휴대폰 대응은 이번 범위 아님(아래 "화면 폭" 참고).

## 불변 1(마스킹) — 화면이 반드시 지키는 것 ★
- **원문(body_raw)·정제본(body_clean)을 화면에 절대 띄우지 않는다.** 화면 본문은 오직 **가려진 참고용(개인정보 마스킹 완료본)** 만 쓴다.
  - 본문 표시 재료: `post_paragraphs.clean_text`(intake가 마스킹해 저장한 문단별 가림본) 또는 `posts.body_pub_ref_path` 파일. **`raw_text` / `body_raw_path` / `body_clean_path` 는 화면 표시에 쓰지 않는다.**
- 마스킹된 자리는 **가려진 표시(예: `[담당자]`, `[가림]`)만** 보이고, **가려지기 전 실제 전화번호·이름은 화면에 다시 띄우지 않는다**(결정 3 참고).
- 이미지: **인물 포함/원본 재사용 금지로 분류된 이미지는 실제 그림을 화면에 띄우지 않는다**(결정 5 참고).

# 1. 씨앗 패턴 (그린필드 — 이후 웹 화면의 기준)

> 색·글꼴은 이미 있는 `templates/brand.css` 의 값을 **그대로 가져다 쓴다**(디자인 규칙: 색은 한 곳에서만). 아래는 그 위에 얹는 **웹 화면용 기본형**이다.

### 색 (전부 brand.css 변수 재사용)
- 주 색상: `--brand`(딥블루) — 상단 띠·제목·링크.
- 강조: `--accent`(틸) — 역할 태그·키워드.
- 본문 글자 `--ink` / 보조 글자 `--muted` / 배경 `--paper`(흰) / 구분선 `--line`.
- **가림 표시 하이라이트**: `--note-bg`(연한 노랑) 배경 + `--note-ink`(갈색) 글자 — "여기가 가려진 자리"를 눈에 띄게.
- **부족한 색(신규 제안)**: brand.css에 "성공/실패/경고" 의미 색이 없다. 품질 판단에 성공·실패·재사용가능/금지 구분이 필요하다 → **결정 1** 참고(사용자 승인 시에만 brand.css에 토큰 추가).

### 글자 크기 (기본형)
- 본문 16px, 작은 보조 13px, 소제목 20px, 화면 제목 24px. 글꼴은 `--font`(맑은 고딕, brand.css) 재사용.

### 간격 단위
- **8px 격자**(8 / 16 / 24 / 32). 모든 여백은 이 배수만 쓴다. (brand.css의 카드 전용 px 값은 카드용이라 웹엔 쓰지 않는다.)

### 링크/버튼 1종
- 읽기 전용이라 무거운 버튼은 없다. **기본형 = 글자 링크**: `--brand` 색, 밑줄은 마우스 올릴 때만. 클릭 영역은 상하 8px 여백 확보(작은 글자만 콕 찍는 링크 금지).

### 상태 배지(pill) 1종 — 재사용 씨앗
- 둥근 알약 모양 + **글자 라벨**(예: `성공`, `실패-삭제된글`). 색은 의미를 **보조**할 뿐, **색만으로 구분하지 않는다**(라벨 글자가 항상 함께). brand.css `card__chip` 개념을 웹 크기로 축소해 재사용.

### 태그 chip 1종 — 재사용 씨앗
- 문단 역할·이미지 분류를 나타내는 작은 라벨. 틸(`--accent`) 계열 + 글자.

> **씨앗 사다리 근거(4단계)**: 기존 `brand.css` 는 인스타 카드(1080×1350 고정 크기) 전용 레이아웃이라 스크롤되는 웹 페이지에 그대로 못 쓴다(body 크기를 카드 크기로 고정함). 그래서 **웹용 레이아웃은 새로 필요**하다 — 단 **색·글꼴은 새로 만들지 않고 brand.css 변수만 참조**한다. 색 공유 방식은 **결정 2**.

# 2. 화면 A — 글 1건 상세 (걸어다니는 뼈대, 먼저 만든다)

## 배치 (위 → 아래)
1. **상단 고정 띠(스크롤해도 붙어 있음)**: 왼쪽 `← 목록으로` 링크 · 가운데 글 제목(길면 한 줄 말줄임) · 오른쪽 **추출 상태 배지**.
   - 이유: 긴 본문을 내려 읽는 중에도 "지금 어떤 글인지 / 돌아가기"가 항상 손에 닿게. (데스크탑이지만 고정 영역 개념 적용.)
2. **글 정보 블록(가로 전체)**: 제목(큰 글자) 아래 한 줄 메타 — `카페명 · 게시판 · 담당 · 작성일 · 조회수(참고 신호) · 길이유형(짧음/보통/긺)`. **조회수 옆에는 반드시 "(참고 신호)"를 붙인다**(불변 3: 성과 아님).
3. **아래 2단 나란히**(데스크탑):
   - **왼쪽(넓게, ≈60%) — "참고용 본문 (개인정보 가림 완료)"**: 문단을 하나씩 세로로. 각 문단마다 [역할 태그] + [📌팩트/📣안내 표시(있을 때만)] + 가림본 문단 글. **가려진 자리는 노랑 하이라이트로 강조.**
   - **오른쪽(≈40%) 위 — "개인정보 가림 결과"**: 가려진 항목을 **종류별 건수**로. 예: `전화번호 2건 · 직원 실명/닉네임 3건 · 오픈채팅 1건`. 총 건수 배지. 0건이면 안내 문구(아래).
   - **오른쪽 아래 — "이미지 분류 (N개)"**: 이미지마다 카드 한 장 — 분류 태그(종류/역할/출처) + **재사용 배지**(가능/권리확인/원본금지) + 인물포함 표시. (그림 표시는 결정 5.)
   - 이유: 가장 흔한 행동은 "본문 훑으며 가림·분류가 맞나 대조"다. 본문(왼)과 가림결과·이미지(오른)를 **나란히** 둬 눈이 오가게.

## 품질 판단을 돕는 시각 장치 ★ (이 화면의 핵심)
- **가려진 자리**: 노랑 하이라이트 + 대괄호 글자(`[담당자]` 등) → 어디가 얼마나 가려졌는지 한눈에. 만약 **가려지지 않은 전화번호가 남아 있으면** 하이라이트 없이 맨숫자로 보이므로 사용자가 즉시 알아챈다(누락 감지가 목적).
- **문단 역할**: 각 문단 앞 태그로 구분(도입/조건설명/절차안내/주의사항/CTA/마무리/배경설명 등).
- **의심 지점 강조**: 역할이 **`배경설명`(자동 분류가 확신 없이 넣은 기본값)** 인 문단은 태그를 **회색 + "확인 필요"** 로 흐리게 표시 → "여긴 기계가 잘 못 나눈 곳"임을 알림. (확신값 상세는 결정 4.)
- **이미지 재사용 배지**: 글자 + 색 — `재사용 가능`(초록 계열) / `권리 확인 필요`(노랑) / `원본 재사용 금지`(빨강 계열) + `인물 포함` 라벨. **색만이 아니라 글자로도** 구분.

## 상태 4종
- **비었을 때(내용 없는 글)**: 추출은 됐으나 문단이 하나도 없는 글 → 본문 자리에 "이 글에는 정리된 본문이 없습니다. (추출은 됐지만 내용이 비어 있어요.)"
- **불러오는 중**: (서버가 바로 그려주므로 대개 안 보이지만) 잠깐이라도 걸리면 "불러오는 중…" 한 줄.
- **실패했을 때** — 두 경우:
  - (a) **추출 실패 글**: 본문·문단 대신 상태 배지 + **실패 사유 그대로**(예: "이 글은 삭제된 글이라 가져오지 못했습니다." / "비공개 게시판이라 접근할 수 없었습니다." / "로그인이 필요한 글이라 가져오지 못했습니다." / "접근할 수 없었습니다(기타)."). 아래에 "이 글은 다시 추출을 돌린 뒤 확인할 수 있어요."(안내만 — 이 화면엔 버튼 없음).
  - (b) **없는 주소로 들어왔을 때**: "그런 글이 없습니다." + `← 목록으로`.
- **성공했을 때**: 위 배치 전체가 정상 표시.

## 문구 (실제 문장 — 그대로 사용)
- 화면/블록 제목: `참고용 본문 (개인정보 가림 완료)` · `개인정보 가림 결과` · `이미지 분류` · `← 목록으로`
- 조회수 라벨: `조회수 (참고 신호)`
- 가림 0건: `가릴 개인정보가 발견되지 않았습니다. (원래 개인정보가 없던 글이거나, 못 잡았을 수 있으니 본문을 함께 확인하세요.)`
- 이미지 0개: `이 글에는 이미지가 없습니다.`
- '배경설명' 태그 보조 문구: `자동 분류 실패 — 확인 필요`
- **금칙 준수**: "성과 등급", "P1/P2/P3" 등 등급/코드 용어를 화면 어디에도 쓰지 않는다. "정제/파싱/렌더링" 같은 코드 용어도 화면 문구에 쓰지 않는다.

## 재사용 목록 / 새로 만드는 것
- 재사용: brand.css 색·글꼴 변수, 씨앗의 상태 배지·태그 chip·링크.
- 새로: 웹용 2단 레이아웃, 문단 스트림, 가림 하이라이트, 이미지 분류 카드 (모두 씨앗 위에서 만듦 — 새 색 없음).

# 3. 화면 B — 목록 (상세 다음에 만든다)

## 배치 (위 → 아래)
1. **상단 띠**: 화면 제목 `추출 글 품질 확인` + 요약 카운트 `총 N건 · 성공 n · 실패 m`.
2. **보기 전환(간단 링크)**: `전체 / 성공만 / 실패만` — 글자 링크. (선택 기능, 최소로.)
3. **글 목록(표)**: 한 줄 = 글 1건. 열: `제목 · 카페 · 상태 배지 · 가림 건수 · 문단 수 · 이미지 수 · 작성일`. **줄 전체가 클릭 영역**(상세로 이동).
   - 가장 흔한 행동 = "이상해 보이는 글 골라 상세로" → 상태 배지와 가림 건수를 열에 노출해 목록에서 바로 의심 글 식별.

## 상태 4종
- **비었을 때(첫 사용)**: `아직 확인할 글이 없습니다. 추출을 먼저 돌린 뒤 이 화면을 새로고침하세요.` (읽기 전용이라 버튼 없이 안내만.)
- **불러오는 중**: `목록을 불러오는 중…`
- **실패했을 때**(자산창고 파일을 못 열 때): `글 목록을 불러오지 못했습니다. 자산창고 파일을 찾을 수 없어요. 자산창고를 먼저 만든 뒤 다시 열어주세요.` (경로·에러코드 등 코드 용어 노출 금지.)
- **성공했을 때**: 표 표시. (개별 글의 추출 실패는 그 줄의 상태 배지로 보이고, 목록 화면 자체는 성공.)

## 품질 시각 장치
- 상태 배지로 성공/실패 즉시 구분(글자+색). **가림 건수 0인 글**은 숫자를 회색으로 흐리게(개인정보가 원래 없었는지 못 잡았는지 확인하라는 신호).

## 재사용 목록
- 상단 띠·상태 배지·태그·링크 씨앗 재사용. 표는 신규(씨앗 위, 새 색 없음).

# 4. 화면 폭 대응 (모바일 체크 대체)
- **데스크탑 우선. 휴대폰 대응은 이번 범위 아님** — 근거: 기준 사용자가 로컬 PC(회사·집)에서 여는 1인용 검수 도구. 휴대폰 사용 시나리오 없음.
- 단, 폭이 좁아져도 **깨지지 않게**: 고정 픽셀 폭으로 표를 잡지 말고 화면 폭에 맞춰 줄어들게. 2단은 폭이 좁으면 위아래로 쌓이게(1단). 긴 본문·긴 제목은 **가로로 삐져나가지 않고 줄바꿈/말줄임**. 이미지 분류 태그도 넘치면 다음 줄로.
- 상단 띠는 **스크롤해도 붙어 있게**(고정 영역).

# 5. 접근성 기본 (단순화 금지)
- 상태·재사용·역할은 **색만으로 구분하지 않는다** — 항상 글자 라벨 동반.
- 클릭 대상(목록 줄·링크)은 손가락/커서가 넉넉히 닿게(작은 글자 단독 링크 금지, 상하 여백 8px 이상).
- 본문 글자 `--ink` on `--paper` 대비 유지. 가림 하이라이트도 글자가 읽히는 대비로.
- 읽기 전용이라 입력칸은 없음(입력 이름표 항목 해당 없음).

# 6. 새 의존성
- 계획의 제약(파이썬 표준 `http.server` + 템플릿) 안에서 **전부 실현 가능**. 데이터는 서버가 미리 읽어 HTML로 그려 보내므로 브라우저 쪽 프레임워크 불필요. 새 웹 프레임워크 들이지 않음.

# 결정 필요 (designer가 정하지 않음 — 사용자 승인 후 implementer 진행)

1. **성공/실패·재사용 의미 색 추가** — 지금 brand.css엔 "성공(초록)/경고/실패(빨강)" 의미 색이 없다. 품질 판단에 성공·실패, 재사용 가능/금지 구분이 자주 필요하다.
   - (권장) brand.css **한 곳에** 의미 색 3개 추가: 성공(초록)·경고(노랑, 기존 note 재사용 가능)·실패/금지(빨강). 색은 여전히 한 곳에서만 관리됨.
   - (대안) 신규 색 없이 기존 틸+노랑+글자 라벨만으로 구분(실패와 경고가 같은 노랑이라 덜 또렷).
   - 장단점: 권장안은 또렷함 ↑ / 색 개수 3개 늘어남. 대안은 색 절제 / 실패·경고 혼동 여지.

2. **브랜드 색·글꼴 공유 방식** — brand.css가 지금 카드 크기(1080×1350)까지 고정해, 웹 페이지가 그대로 쓰면 화면이 카드 크기로 깨진다. (색은 계속 한 곳에서 관리한다는 원칙은 유지하되) 방식 택1:
   - (권장) 색·글꼴 변수만 별도 작은 파일로 떼고, 카드용 크기 규칙은 카드 쪽에만 남긴다.
   - (대안) 카드 크기 규칙을 카드 묶음 안으로만 한정하고 파일은 그대로 둔다.
   - (사용자 관점 요약: 어느 쪽이든 "색은 여전히 한 곳에서만 바꾸면 전체 반영".)

3. **가림 항목을 어떻게 보여줄까** — 가려지기 전 실제 값(전화번호·이름 원본)을 화면에 같이 보여줄지.
   - (권장, 불변 1 부합) **종류+건수만** 보여주고 원본은 화면에 다시 안 띄운다. 본문에서 가림 자리는 하이라이트로 확인, 누락은 맨숫자로 드러나므로 검수 가능.
   - (대안) 검수 편의로 원본도 잠깐 보여줌 → **개인정보를 화면에 재노출**하게 되어 불변 1 취지와 충돌. 비권장.
   - 참고(구현 메모): 가림 항목(종류·건수)은 지금 DB에 저장돼 있지 않다. 화면에 보이려면 뷰어가 가림을 다시 계산하거나, 저장 항목에 추가해야 한다 — 방식은 implementer 판단(어느 쪽이든 원본은 화면에 안 띄움).

4. **문단 '확신 낮음' 강조 방식** — 자동 분류의 확신값이 지금 저장돼 있지 않다.
   - (권장) 우선 **`배경설명`(확신 없이 넣는 기본값) 문단만 "확인 필요"로 흐리게** 표시 — 추가 저장 없이 가능.
   - (대안) 확신값을 저장 항목에 추가해 도입/마무리 등도 "확신 낮음"을 표시(더 정확하나 저장 항목 추가 필요 — planner/implementer 판단).

5. **이미지 실제 그림 표시 여부** — 분류 결과와 함께 실제 그림 썸네일을 보여줄지. **인물 포함·캡처·원본 재사용 금지 이미지는 개인정보 노출 위험**(불변 1: 개인정보 든 이미지 원본 재사용 금지).
   - (권장, 불변 1 부합) `인물 포함=예` 또는 `원본 재사용 금지`로 분류된 이미지는 **그림 대신 회색 자리표시 + "미리보기 가림(개인정보 가능성)"** 문구만. 그 외(안전 분류)만 썸네일 표시.
   - (대안) 전부 썸네일 → 검수는 쉬우나 개인정보 이미지가 화면에 노출될 수 있음. 비권장.
   - (더 보수적 대안) 이번엔 **썸네일 아예 없이 분류 정보만** — 가장 안전, 대신 이미지 분류가 맞는지 눈으로 대조하긴 어려움.
