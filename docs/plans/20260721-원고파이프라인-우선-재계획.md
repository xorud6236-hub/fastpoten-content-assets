## 1. 판정 — 현재 결정 8(축소·재초점)과 양립하는가

**대부분 양립한다. 뒤집는 것은 딱 한 조각, '학습'이다.**

축소 결정 문서(§재초점 후 최소 가치 조각)가 "지을 것"으로 못 박은 4개 중 3개가 이번 요청과 같은 것이다.

| 사용자가 말한 것 | 축소 결정에서의 위치 | 판정 |
|---|---|---|
| AI 글작성 | "③ 팩트 채운 원고 초안 도우미 — 진짜 수치 들어간 초안" | **이미 지을 것으로 정해져 있음.** 뒤집기 아님 |
| 검토·수정 | 사람이 하는 일(도구 아님) | 양립 |
| 수동발행 | "**발행은 손으로**" 명시 | 양립. 불변 7·8과도 일치 |
| **학습** | 5층 자기 개선 루프 = **보류**. 근거는 자사 채널 실측만(불변 3) | ★ **여기만 충돌 가능** |
| 글↔룰북 더블체크 | 자체 구축 정당 3개 중 "룰북 팩트 강제" | 양립(오히려 급소 해소) |

**'학습'을 둘로 갈라야 한다.**
- (가) **내가 고친 흔적에서 배우기** — 내가 AI 초안을 어떻게 고쳤는지를 모아 다음 초안에 예시로 붙이는 것. 이건 성과 데이터가 아니라 **내 취향·문체 기록**이므로 불변 3에 걸리지 않는다. 그리고 가장 작은 형태(= 고친 원고를 그냥 모아두고 다음에 붙여넣기)면 시스템이랄 것도 없다.
- (나) **조회수 좋은 글을 따라 배우기** — 카페 조회수는 참고 신호일 뿐 성과가 아니다(불변 3). 자사 채널(워드프레스·홈페이지) 실측은 아직 없다. **이걸 하겠다면 결정 8과 불변 3을 동시에 건드리는 것이고, 지금은 근거가 없다.** 게다가 실측 결과 "잘 되는 글은 형식이 아니라 주제·시의성이 가른다"(FEEDBACK_LOG 2026-07-13) — 형식을 학습해봐야 얻을 게 없다는 것도 이미 확인됐다.

→ 권장: **(가)만, 그것도 자동 학습이 아니라 "고친 원고를 모아두고 다음 초안에 예시로 붙이기"까지.** (나)는 워드프레스 트래픽이 생긴 뒤(결정 8의 재개 조건).

**한 가지 더 정직하게 — "빠르게"가 이 프로젝트를 시험한다.**
축소 결정의 kill 기준에 이런 문장이 있다: *"원고 1건 팩토리 소요시간 > 클로드 직접 붙여넣기 시간 → 정착 불가 신호."* 지금 가장 빠른 방법은 사실 **클로드 창에 직접 쓰는 것**이다. 그러므로 이번에 자체로 지을 것은 문장 생성기가 아니라 **"AI에게 넘길 재료를 한 번에 모아주는 것"**이다. 재료를 모으는 일(우리 검증된 글 + 확인된 팩트 + 금지어 + 가림 규칙)은 클로드가 대신 못 한다 — 그게 창고를 만든 이유다.

## 2. 가장 작은 끝-끝 (기성품 / 자체 구축 구분)

```
[주제 고르기]  ← 창고(이미 있음: 트렌드·주제 필터)
   ↓
[재료 꾸러미 한 장]  ← ★ 자체 구축 (여기만)
   확인된 팩트 + 미확인이면 [검수:] 딱지 + 우리 잘 읽힌 글 3~5건 + 금지어 + 지킬 규칙
   ↓ 복사
[초안 만들기]  ← 기성품 (지금 쓰는 클로드 창. 추가비용 0)
   ↓
[검토·수정]  ← 사람 (클로드 대화에서 그대로)
   ↓
[완성본 보관]  ← ★ 자체 구축 (붙여넣기 1칸) = '학습'의 최소 형태
   ↓
[발행]  ← 손으로 (워드프레스). 불변 7·8
```

- **사서 쓰는 것**: 문장 생성(클로드/GPT), 워드프레스 편집·발행, 성과 수집(서치콘솔·GA 무료).
- **자체 구축이 정당한 것**: 재료 꾸러미(창고+룰북+마스킹이 있어야만 만들어짐), 완성본 보관.
- **지금 안 만드는 것**: 버튼 하나로 자동 생성(AI API 연결) — 위 흐름을 실제로 몇 편 돌려보고 "복사·붙여넣기가 진짜 병목"으로 확인된 뒤에.

## 3. ★ 선행 조건 판정 — CA-6 4차(팩트 고치는 화면)는 원고보다 먼저다

**그렇다. 우회로가 없다.**
- 팩트 51건이 **전부 '미확인'**이고, CA-6 계획 §7에서 불변 6이 *"룰북에 있어도 미확인이면 `[검수:]`"*로 확장 확정됐다. → 지금 원고 도우미를 만들면 **나오는 게 전부 `[검수:]`**다. 그건 축소 결정의 kill 기준("팩트로 채워지는 비율이 낮으면 차별점 없음")에 그대로 걸린다.
- 엑셀에서 고쳐 다시 넣는 우회도 막혀 있다 — 적재기가 **추가 전용**이라 기존 항목을 덮지 않는다(D1 결정). **화면이 유일한 수정 경로다.**
- 다만 **51건 전부를 검수할 필요는 없다.** 첫 원고 주제(사회복지 8건)만 확인하면 뼈대가 돈다.

## 5. (1)번 더블체크 — 무엇을 무엇과 대조하나

**원칙 먼저: 글에서 팩트를 가져오지 않는다.** 글은 직원이 쓰고 검수한 **두 번째 의견**일 뿐이고, 룰북과 마찬가지로 틀릴 수 있다. 기계는 **어긋난 곳을 발견해 나란히 보여주기만** 하고, 어느 쪽이 맞는지는 사용자가 판정한다(불변 6 유지).

### 5-1. 이름이 안 맞는 문제 (실측: 팩트 51 중 17건이 글 0건, 그중 최소 7건 오탐)

자동 이름 맞추기로는 못 푼다. AI로 추론해 자동 연결하면 **조용한 오답**이 되고, 그건 이 프로젝트가 여러 번 데인 실패 방식이다(FEEDBACK_LOG: "무해를 증명하려면 추측한 후보가 아니라 전체를 돌려야 한다").

**해법: 사람이 한 번 이어준다(51번).** 팩트 상세 화면에 '이 팩트와 이어진 주제' 칸을 두고, 기계는 **후보만 추천**(이름 부분일치 + 글 많은 주제 순)하고 사용자가 체크로 고른다. 한 팩트에 주제 여러 개를 이을 수 있다(편입학 ↔ 방통대·사이버대).
이 연결표는 **더블체크와 원고 재료 꾸러미 양쪽이 같이 쓴다** — 한 번 일해서 두 곳에 쓰인다. 새 테이블을 만드는 이유가 이것이다(기존 `keywords.category`는 룰북 카테고리 15종용이라 팩트 51건 단위를 못 담는다).

### 5-2. 무엇을 대조하나 (가장 작은 형태)

**숫자만 본다.** 팩트 칸에서 `숫자 + 단위`(시간·학점·개월·년·급·원)를 뽑고, 이어진 주제의 **참고용 본문(가려진 것)**에서 같은 단위의 숫자가 든 문장을 뽑아 **값이 다른 것만** 나란히 보여준다.

예시(실제 검수 1호):
```
사회복지사 2급 · 실습시간
  룰북 요건란 : 160시간         [미확인]
  룰북 FAQ    : 120시간 필수    [미확인]   ← 룰북 안에서 이미 어긋남
  우리 글 12건 : 160시간 (9건) / 120시간 (3건)   ← 글 제목·링크 함께
```
- 문장 전체를 자연어로 비교하지 않는다(그건 AI 호출·비용·오판). 숫자 불일치만으로도 검수 1호가 그대로 잡힌다.
- 화면에는 **근거를 감추지 않는다**(헌장 디자인 규칙) — 어느 글의 어느 문장인지 눌러서 갈 수 있어야 한다.
- 판정 결과를 저장하지 않는다(읽기 전용). 사용자가 팩트를 고치면 그걸로 끝 — 별도 '판정 상태'는 지금 필요 없다.
- 성능: 글목록 교훈(현재 결정 10)대로 화면이 매번 3,777건을 훑게 두지 않는다. **팩트 1건 상세에서 이어진 주제의 글만** 연다(수십~수백건). 그래도 느리면 그때 세서 저장한다.

이유: 3차까지가 **가장 얇은 끝-끝**이다. 5·6차(이어주기·더블체크)는 훌륭한 도구지만 **첫 원고에는 필요 없다** — 51건 전체 검수를 빠르게 하는 도구다. 뼈대가 돌기 전에 살을 붙이지 않는다. 다만 더블체크를 먼저 보고 싶다면 순서를 바꿔도 무방하다(결정 Q6).

배포 개념 없음(로컬 실행). 데이터 구조 변경 차수(4·5)는 각각 단독으로 분리했고, 실행 전 창고 파일 사본 1부를 둔다.

## 9. 불변 준수 확인

- 불변 1: 완성 원고 저장(4차)·재료 꾸러미 출력 모두 `masking.py` 통과. 더블체크는 **참고용(가려진) 본문**만 읽는다.
- 불변 3: '학습'은 (가)로 한정 — 카페 조회수를 성과로 쓰지 않는다. 꾸러미의 조회수 표시는 "참고 신호" 용어 유지.
- 불변 4: 본문은 파일에만. 완성 원고는 **우리가 새로 쓴 글**이라 posts와 별개로 보관(원문 건드리지 않음).
- 불변 6(+CA-6 §7 확장): 미확인 팩트는 꾸러미에서 `[검수:]`로 표시하고 수치를 단정하지 않는다.
- 불변 7·8: 발행은 사람이 손으로. 이번 계획에 자동 발행 없음.
- 불변 9: 새 테이블은 멱등 ensure, 컬럼 삭제 없음.
- 헌장 디자인 규칙: 더블체크·꾸러미 모두 **근거(원글·원본 칸)로 갈 수 있는 길**을 같은 화면에 둔다. 화면의 숫자는 무슨 숫자인지 화면이 말한다.
