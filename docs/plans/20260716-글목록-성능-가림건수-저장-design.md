## 0. 한 줄

지금 화면에서 **바뀌는 곳은 두 군데뿐**이다 — 목록 바로 위에 "어디까지 보는 중인지" 한 줄, 목록 바로 아래에 "쪽 이동" 한 줄. 그리고 '가림' 칸에 숫자 대신 **"다시 세기 필요"** 배지가 뜰 수 있게 된다. **열·필터·정렬·상단 배지·행 생김새는 한 글자도 안 바뀐다.**

## 1. 간결 사다리 — 새로 만드는 게 왜 이것뿐인가

| 필요한 것 | 사다리 | 결론 |
|---|---|---|
| "다시 세기 필요" 표시 | 1단(기존 패턴) | **상태 칸의 배지와 같은 패턴**(`badge warn`). 사용자가 이미 배운 모양. 새로 안 만듦. |
| 범위 안내 한 줄 | 1단 | 분석 화면의 "상위 50건만 보입니다 (전체 N건)" 줄과 **같은 패턴·같은 자리**(목록 바로 위, 회색 작은 글씨). |
| 빈 상태 | 1단 | 기존 `state` 안내 상자 그대로. |
| 누를 수 없는 쪽 이동(첫 쪽의 '이전') | 2단(기존 토큰 조합) | 조회수 없는 글의 `-` 표기에 쓰는 **흐린 글자색**을 그대로. |
| 쪽 이동 줄 | **4단(새 패턴 1개)** | 필터 줄의 작은 글자 링크를 그대로 쓰면 **손가락으로 누르기엔 작다**. 필터는 가끔 한 번 누르지만 **쪽 이동은 훑는 동안 반복해서 누르는 주 동작**이라 누를 수 있는 크기가 필요하다. → 쪽 이동 줄만 링크에 여백을 줘 **손가락 크기(약 44px 높이)** 확보. **색·글꼴은 기존 것만 조합, 새 색 없음.** |

**`tokens.css`에 새 색을 넣지 않는다.** 필요한 색(주색·흐린 글자·경고 노랑)이 이미 다 있다.

## 2. 배치 (위 → 아래) — ★는 이번에 새로 생기는 줄

```
┌───────────────────────────────────────────────────────────────────┐
│ [상단 띠]  글 목록 · 분석 · 트렌드 · 주제 검수 · 데이터            │  그대로
│            추출 글 품질 확인        [총 3,777건 · 성공 3,621 · 실패 156] │  그대로(창고 전체 기준)
└───────────────────────────────────────────────────────────────────┘
  보기: 전체 · 성공만 · 실패만  |  정렬: 최신순 · 조회수 높은 순     ← 그대로
                                     (조회수는 참고 신호입니다.)

★ 가림 건수를 다시 세야 하는 글이 15건 있습니다. …            ← ①안내(있을 때만)
★ 3,777건 중 101~200번째 보는 중 · 2 / 38쪽                    ← ②범위(한 쪽이면 숨김)

  제목        │ 카페 │ 담당자 │ 상태 │  가림   │ 문단 │ 이미지 │ 조회수 │ 작성일   ← 그대로
  ───────────────────────────────────────────────────────────────
  임상심리사…  │공준모│ 김OO  │[추출완료]│  3건    │  8  │   6   │   456 │2025-03-02
  장기요양…    │소감아│ 이OO  │[추출완료]│[다시 세기 필요]│ 5 │ 2 │ 1,204 │2025-02-11  ★
  …  (한 쪽에 100줄)
  ───────────────────────────────────────────────────────────────

★  ← 처음 · ← 이전 ·  2 / 38쪽  · 다음 → · 끝 →                  ← ③쪽 이동
```

**가장 흔한 동작이 가장 가까운가**: 이 화면의 주 동작은 ①줄 하나 눌러 상세 보기(→ 줄 전체가 링크, 그대로) ②100줄을 다 훑고 **다음 쪽으로** 넘어가기. ②는 훑기가 끝나는 자리 = **목록 맨 아래**에 둔다. 위에는 이동 단추를 겹쳐 놓지 않고 **범위 한 줄만**(지금 어디인지 알기 위함) — 같은 단추를 위아래에 두 번 두면 화면만 시끄럽고 최소 변경 원칙에 어긋난다. (다르게 하고 싶으면 결정 2)

## 3. "총 N건"이 헷갈리지 않게 하는 법 ★

상단 배지는 **창고 전체 기준**이고 손대지 않는다. 대신 목록 바로 위 ②줄이 **같은 숫자를 다시 부르며** 범위를 말하게 해서, 두 줄을 나란히 읽으면 저절로 이어지게 한다.

| 지금 보기 | 상단 배지(그대로) | ② 범위 줄(신설) |
|---|---|---|
| 전체 · 2쪽 | 총 3,777건 · 성공 3,621 · 실패 156 | `3,777건 중 101~200번째 보는 중 · 2 / 38쪽` |
| 성공만 · 1쪽 | 총 3,777건 · 성공 3,621 · 실패 156 | `성공만 3,621건 중 1~100번째 보는 중 · 1 / 37쪽` |
| 실패만 · 2쪽 | 총 3,777건 · 성공 3,621 · 실패 156 | `실패만 156건 중 101~156번째 보는 중 · 2 / 2쪽` |
| 결과가 한 쪽뿐 | 총 3,777건 · 성공 3,621 · 실패 156 | `156건 모두 보는 중` (③ 쪽 이동 줄은 **아예 숨김**) |

"보이는 범위 + 전체 건수"를 목록 위에 적는 건 이 뷰어의 기존 습관이다(분석 `상위 50건만 보입니다 (전체 690건)`, 주제 검수 `상위 N개 (전체 1,327개)`). **같은 자리·같은 회색 작은 글씨**로 맞춘다. → 이게 세 번째라서 §8에 규칙 승격을 제안한다.

## 4. 쪽 이동 줄 — 생김새와 규칙

```
첫 쪽:    처음   이전   1 / 38쪽   다음 →   끝 →
                (흐림) (흐림)
가운데:  ← 처음 · ← 이전 ·  2 / 38쪽 · 다음 → · 끝 →
끝 쪽:   ← 처음 · ← 이전 · 38 / 38쪽   다음    끝
                                       (흐림) (흐림)
```

- **누를 수 없을 때 없애지 않고 흐리게 남긴다** — 없애면 남은 단추 위치가 쪽마다 움직여서, 같은 자리를 연달아 누르는 훑기가 방해받는다.
- **흐림은 색만으로 구분하지 않는다**: 화살표(`←`/`→`)와 밑줄이 함께 사라지고, 눌리지 않는다. 흑백으로 봐도 "이건 링크가 아니다"가 보인다.
- 번호를 1부터 38까지 늘어놓지 않는다(결정 1). 이 화면의 훑기는 **순서대로 넘기기**이고, 조회수순에서 중요한 건 앞쪽이라 '처음'만 있으면 충분하다. 특정 쪽으로 바로 가고 싶으면 주소창의 쪽 번호를 고치면 된다(사용자가 이미 하는 방식은 아니므로 필요해지면 그때 늘린다 — YAGNI).
- **쪽을 옮겨도 지금 보기·정렬이 그대로 따라온다.** 반대로 **보기·정렬을 바꾸면 무조건 1쪽부터**(다른 목록이 됐는데 27쪽에 서 있으면 텅 빈 화면이 뜬다).
- 없는 쪽(0쪽·99쪽·글자)을 주소로 넣으면 **오류 화면 대신 가장 가까운 쪽**(1쪽 또는 끝 쪽)을 보여준다. 기존 보기·정렬 값과 같은 방식(허용된 것만 통과).

## 5. '가림' 칸의 상태 — 문구와 표현

| 창고 상태 | 칸에 보이는 것 |
|---|---|
| 지금 규칙으로 세어 둔 글 | `3건` (지금과 동일) |
| 세어 봤는데 0건인 글 | `0건` (흐린 글자 — 지금과 동일) |
| **가림 규칙이 바뀐 뒤 아직 안 센 글** | **`다시 세기 필요`** (노란 경고 배지) |
| **아예 한 번도 안 센 글** | **`다시 세기 필요`** (같은 표현) |

**두 경우를 한 문구로 덮는 게 맞다.** 사용자가 할 일이 똑같기 때문이다(다시 세라고 시키기). "아직 안 셈"과 "옛 숫자"를 구별해 보여주면 화면만 복잡해지고, 사용자가 그 차이로 다르게 행동할 일이 없다.

- 표현은 **상태 칸 배지와 같은 패턴**(글자 + 테두리 + 배경). **색만으로 구분하지 않는다** — "다시 세기 필요"라는 글자 자체가 뜻이다.
- 이 배지가 들어가려면 '가림' 칸이 지금보다 넓어야 한다. **가림 칸을 넓히고 그만큼 제목 칸을 줄인다**(제목은 지금도 말줄임 처리라 안전). 나머지 열 너비는 그대로.
  - 지금: 제목 2.6 / 카페 1 / 담당자 1 / 상태 1.2 / **가림 0.7** / 문단 0.6 / 이미지 0.7 / 조회수 0.9 / 작성일 1.1
  - 바꿈: 제목 **2.0** / 카페 1 / 담당자 1 / 상태 1.2 / **가림 1.3** / 문단 0.6 / 이미지 0.7 / 조회수 0.9 / 작성일 1.1 (합계 동일)

## 6. 문구 (실제 문장 — 그대로 사용)

**① 안내 줄** — 이 상태인 글이 **1건이라도 있을 때만** 보인다. 없으면 줄 자체가 없다.
> 가림 건수를 다시 세야 하는 글이 15건 있습니다(가림 규칙이 바뀌었거나, 아직 세지 않은 글입니다). 클로드 코드에 "가림 건수 다시 세 줘"라고 말하면 정리됩니다.

- 건수는 **지금 이 쪽이 아니라 전체 기준**(그래야 "15건" = 할 일의 크기가 된다).
- 왜 숫자를 안 보여주는지까지 말한다 — 이 화면이 못 보여주는 게 아니라 **틀린 숫자를 안 보여주는 것**이다.
- 명령 이름(파일명)은 화면에 쓰지 않는다(결정 5).

**② 범위 줄** — §3 표 그대로.

**③ 쪽 이동 줄** — `← 처음` / `← 이전` / `2 / 38쪽` / `다음 →` / `끝 →`

**칸 안** — `다시 세기 필요`

**빈 상태(거른 결과가 0건일 때)** — 지금은 표 머리만 덩그러니 남는데, 이번에 채운다.
> 실패만 보기에 해당하는 글이 없습니다. 위에서 '전체'를 눌러 보세요.
> 성공만 보기에 해당하는 글이 없습니다. 위에서 '전체'를 눌러 보세요.

**빈 상태(창고에 글이 아예 없을 때)** — 지금 문장 그대로.
> 아직 확인할 글이 없습니다. 추출을 먼저 돌린 뒤 이 화면을 새로고침하세요.

**실패했을 때(창고를 못 열 때)** — 지금 문장 그대로.
> 글 목록을 불러오지 못했습니다. 자산창고 파일을 찾을 수 없어요. 자산창고를 먼저 만든 뒤 다시 열어주세요.

## 7. 상태 4종

| 상태 | 이 화면에서 |
|---|---|
| **비었을 때** | 위 두 가지 빈 상태 문구(창고가 빔 / 거른 결과가 0건). 둘 다 "다음에 뭘 하면 되는지"까지 말한다. |
| **불러오는 중** | **따로 만들지 않는다**(사다리 3단 — 브라우저 기본 동작). 지금은 36초라 필요했을 수 있지만 이 차수의 목표가 **1초 이내**다. 1초짜리 화면에 로딩 표시를 넣으면 깜빡임만 늘어난다. 1초를 못 지키면 로딩 표시가 아니라 성능을 다시 본다. |
| **실패했을 때** | 지금 문구 그대로(§6). |
| **성공했을 때** | §2 배치. 그중 '가림' 칸만 글마다 `N건` 또는 `다시 세기 필요`. |

## 8. ★ 반복 수정 감지 — 규칙 승격 제안 (반영은 사용자만)

**"목록 일부만 보임 + 전체 건수 알려주기"가 이번이 세 번째다.** 분석 화면(`상위 50건만 보입니다 (전체 N건)`), 주제 검수 화면(`상위 N개 (전체 1,327개)`), 그리고 이 화면. 세 번 다 그때그때 문장을 새로 지었다. 네 번째가 생기기 전에 규칙으로 올리기를 제안한다.

> **헌장 §디자인 규칙에 넣을 문안(초안)**
> - 목록이 전체 중 일부만 보일 때는 **목록 바로 위에 회색 작은 글씨 한 줄**로 "보이는 범위 + 전체 건수"를 적는다(예: `3,777건 중 101~200번째 보는 중 · 2 / 38쪽`). 상단 배지의 건수는 **항상 창고 전체 기준**이며, 화면에 보이는 줄 수를 뜻하지 않는다.

넣을까요? (designer는 제안만 하고, 헌장은 사용자만 고칩니다.)

## 9. 모바일 체크 (실사용은 PC — 헌장 규칙상 확인)

- **엄지 구역**: 주 동작인 '다음 쪽'이 **목록 맨 아래** = 한 손 엄지가 닿는 자리. 맞다.
- **터치 크기**: 쪽 이동 링크만 **손가락 크기(약 44px 높이)** 확보(§1의 새 패턴 근거). 필터 줄은 지금 크기 유지(이번 과제 밖 — 최소 변경).
- **키패드**: 이 화면엔 입력칸이 없다. 해당 없음.
- **넘침**: 표 9칸은 **지금도 좁은 화면에서 눌려 보인다 — 이번에 손대지 않는다**(기존 문제, 최소 변경). 다만 **쪽 이동 줄은 좁으면 두 줄로 자연스럽게 넘어가게** 한다(가로로 삐져나가지 않음). 안내 줄·범위 줄도 줄바꿈 허용.
- **고정 영역**: 없음. 상단 띠는 지금도 붙어 있고, 쪽 이동을 아래에 고정하면 100줄 훑기 내내 화면을 가린다 — 하지 않는다.

## 10. 접근성 기본 (단순화 금지)

- 쪽 이동은 **글자 링크**(주소가 있는 진짜 링크) — 키보드로 넘길 수 있고, 새 탭으로 열 수 있고, 뒤로 가기가 동작한다. JS 없음(기존 씨앗 유지).
- 누를 수 없는 쪽 이동은 **흐린 색 + 화살표 없음 + 밑줄 없음**(색만으로 구분 안 함).
- '다시 세기 필요'는 **글자로 뜻을 말한다**(노란색은 거들 뿐).
- '가림' 칸의 `N건`·`다시 세기 필요`는 **같은 열의 다른 상태**임이 열 제목('가림')으로 이어진다.

## 11. 재사용 목록
