# 축소·재초점 결정 (2026-07-07) — field-critic + scout 종합

> 사용자 질문 "이 프로젝트가 왜 필요한지, 실제 쓸 이유가 있는지"에 대한 적대적 검토 결과와 결정.

## 판정: 축소·재초점 (사용자 채택)
5층 전체를 지금 짓는 것 보류. 중단 아님(창고는 진짜 자산). 이길 근거가 검증 안 된 층에 1인 비개발자의 유지보수를 미리 떠안기지 않는다.

## 채널 현실 (사용자 확인)
- **자사 홈페이지: 유입 꽤 있음** → 콘텐츠를 쓸 실독자 채널이 이미 존재(발행이 헛되지 않음).
- **워드프레스: 테스트만, 거의 0 — 이제 키우는 중** → 자동발행부터 짓는 건 빈 채널에 쏘는 것. 3층 보류 타당.

## 자체 구축이 정당한 것 = 3개뿐 (시중 도구가 대체 불가)
1. **재료 창고(0층)** — 마스킹·중복제거된 검증 콘텐츠 1만건 + 참고신호. 시장 대응품 없음. **이 프로젝트의 심장.**
2. **우리말·우리 바닥 개인정보 마스킹** — 오픈채팅·OO쌤·카페닉네임·직원실명. 영어권 기성품이 못 잡음(불변 1).
3. **룰북 팩트 강제** — 학점·자격 수치 틀리면 리스크 → 없으면 [검수:] 강제(불변 6). 단, "완전 커스텀"까지 안 가고 "프롬프트+금지어"로도 상당 커버 가능(재검토 여지).

## 사서 쓸 것 (지금 자체 구축 금지 — 후회 위험)
- 원고 SEO 문장 생성(Koala·Jasper류), 워드프레스 자동발행(Koala 플러그인/Machined), 인스타 예약(Buffer·Mirr), **성과 수집(서치콘솔·GA·인스타 인사이트 — 무료로 이미 다 줌)**.

## 지금 가장 아픈 급소
1. **우리만의 무기(룰북 팩트)가 비어 있음** — 지금 원고는 [검수:] 투성이 → 범용 AI와 차이 없음. **최우선 해금.**
2. 물량 8편/4주 → 손+클로드가 이김. 자동발행 설비는 과잉(잘 고장나고 1인이 유지).
3. 개선루프(5층)는 실측 데이터 몇 달치 쌓이기 전엔 근거 없음(불변 3). 데이터 먼저, 루프 나중.

## 재초점 후 최소 가치 조각 (지을 것)
1. **룰북 팩트 시트 적재(CA-6 앞당김)** — 무기 해금. 엑셀 ②팩트 시트 → DB.
2. **재료 창고 검색** — "이 주제로 잘 됐던 우리 글 찾아줘"(창고+뷰어 거의 됨).
3. **팩트 채운 원고 초안 도우미** — 진짜 수치 들어간 초안. **발행은 손으로.**
4. 카드: Canva로 시작해도 되는 수준(품질 병목이 폰트·아이콘). 자체 렌더 고도화는 물량 나오면.

## 3·4·5층 재개 조건 (kill/go 기준)
