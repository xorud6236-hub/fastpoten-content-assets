# 구조 DNA 샘플링 — 방향·결정·진행 (2026-07-09)

> 세션 재시작 후 이 노트부터 읽고 이어간다. B 경로(콘텐츠 구조를 배워 원고 생성)의 앞단.

## 목표
"잘 되는 우리 글의 **구조 DNA**"를 데이터에서 발견 → 그 틀로 원고를 뽑는다. (우리가 6칸을 정하는 게 아니라 데이터가 틀을 알려줌 + 참고신호와 상관.)

## 핵심 결정
- **"잘 되는 글" 기준 = 기간대비 조회수(하루당 = 조회수 ÷ 게시 경과일).** 순수 조회수는 오래된 글 유리(누적)라 보정.
  - caveat: 일당 조회수는 최근글 유리(조회수는 초반에 몰림). 추후 "급상승 vs 스테디" 구분으로 정밀화.
- **순위(rank)는 선택 기준에서 제외 — 데이터 신뢰 불가.** 근거(실측):
  - rank_value ↔ exposure_status 불일치: `2p 1등→value=2 Top3`, `1p 1등→value=17 Top30` 등. **페이지가 순위값에 안 붙음.**
  - 소스 혼재: 통합탭 변동순위 / 순위(view통합) / 카페탭순위가 섞임. exposure_status는 2,453/10,118만 채움.
  - → Top3 버킷으로 "잘 된 글"을 고르면 2페이지짜리·엉뚱한 글이 섞인다.
- **조회수는 엑셀에 없음** → 글을 추출해야 확보(카페 페이지에서 뜯음). 그래서 순위로 미리 못 거르니 **추출 후 조회수로 판별**.

## AI 태깅 3단계 + 비용 (중요)
- **A. 패턴 발견**(샘플 수백 건 → 반복 구조·유형 추출): 어려운 추론=Opus. **Claude Code(Max 플랜)로 이 자리에서 = 추가비용 0.**
- **B. 전체 태깅**(발견한 틀로 전 글 라벨): 단순 적용=Haiku/Sonnet + Batch API. **API 별도 결제**(Max와 별개). 공준모 3,871건 태깅 ≈ Haiku $7 / Opus $33 (Batch).
- **C. 조회수↔순위 상관**: 코드 계산, LLM 비용 ~0.
- ★ **Max 플랜 ≠ API.** 대화형 발견(A)은 Max로 무료, 대량 프로그램 태깅(B)만 API 별도 결제.
