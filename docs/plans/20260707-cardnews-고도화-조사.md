# 카드뉴스 고도화 조사 (2026-07-07, scout) — 기록용, 착수 보류

> 사용자 판단: 카드 고도화는 기록만. 프로젝트 근본 가치 재검토 후 재개.

## 한 줄 진단
정보 정렬·팩트 출처·마스킹은 이미 프로급인데, **"저장하고 싶은 느낌"을 만드는 3요소(폰트 인상·시각화·시리즈 구조)가 비어** 아마추어 인상. **브랜드 색/로고 미정은 병목이 아님**(tokens.css 한 곳 교체로 전 카드 반영 — 나중에 확정해도 재작업 0).

## 필요한 것 우선순위 (효과/난이도)
1. **폰트 교체**(맑은고딕 → Pretendard 등 상업용 무료 고딕) — 효과 큼·난이도 낮음. headless Chrome이 로컬 폰트 불러오게 1회 세팅. 체감 변화 최대.
2. **시각화**(행마다 아이콘/번호) — 효과 큼·난이도 중. 오픈소스 아이콘셋(Lucide·Tabler·Phosphor류 SVG) 내장. 스타일 1종 통일 필수.
3. **캐러셀 구조**(표지→핵심→요약 3장) — 효과 큼(저장·도달)·난이도 중. 템플릿 1종→3종. 표지는 "후킹"만(정보 욱여넣기 금지).
4. **비교/단계 카드 템플릿 추가**(CB-2 원계획과 일치) — 효과 중.
5. **브랜드 색 확정** — 효과 중·결정만. tokens.css 교체 대응이라 대기 불필요.
6. **로고 삽입** — 효과 소~중. 없어도 브랜드명 텍스트로 굴러감. (현재: 로고 없음)

## 함정 (즉시 아마추어 신호)
한 카드에 정보 욱여넣기 / 아이콘 스타일 혼용 / 색 3개 초과 / 글자 그림자·외곽선 / "로고 확정까지 전체 대기".

## 우리 방식 vs Canva (헌장 결정 4 "손발은 사서 쓴다")
- **우리 방식(AI=JSON, 템플릿 렌더)이 유리**: 룰북 팩트 대량 생산 + 마스킹·팩트출처·금지어를 구조로 자동 강제(사람이 Canva로 하면 매번 실수 지점). → 정보성 카드 대량 생산은 우리 방식.
- **Canva가 나은 지점**: 자유 일러스트·사진 합성·감성 표지 1회성/캠페인 1장. 용도 분리(정보=우리, 감성표지=Canva). 아이콘셋 내장 시 우리 커버 범위 확대.

## 가장 큰 레버 2개 (재개 시 결정)
- 레버1: 폰트 교체(가장 싸고 체감 큼, 색·로고 무관 착수 가능).
- 레버2: 단장 1장 vs 캐러셀 3장(저장·도달 엔진, 콘텐츠 전략 본체).
