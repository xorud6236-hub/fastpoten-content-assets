## 1. YAGNI — 이 작업이 정말 필요한가, 어디까지가 최소인가

**필요하다**(축소 결정 8이 "지을 것"으로 남긴 "팩트 채운 원고 초안 도우미"가 이것). 단 최소 형태를 지킨다:

- **자체 구축은 세 조각뿐**: ①재료 조립(창고+룰북+마스킹이 있어야만 만들어짐 = API에 넘길 입력) ②API 호출·마스킹·저장 ③화면 편집기 + 완성본 보관. 문장 생성은 기성품(API), 발행은 손.
- **새 의존성 판단**: Claude API 호출에 SDK(`anthropic` 패키지)를 새로 넣을지 vs 표준 라이브러리(`urllib`)로 HTTPS POST를 직접 보낼지는 구현 판단. 뷰어가 이미 stdlib http.server만 쓰는 정신(현재 결정 없음, PC 이동 시 안 깨짐)을 고려하면 **표준 라이브러리 직접 호출이 기본, SDK는 재시도·에러 처리가 눈에 띄게 번거로울 때만**. (이유를 대지 못하면 새 패키지 안 넣음.)
- **새 파일**: 재료 조립 + API 호출은 뷰어 요청 처리 안에 넣지 않고 **별도 모듈**로 뺀다. 이유: API 호출은 수 초~수십 초 걸리고 실패·비용이 있어, 읽기 전용 뷰어의 요청 핸들러 안에서 블로킹하면 화면이 멈추고 취약해진다(선례: bulk_extract도 배치를 뷰어 밖에 둠).

## 2. 가장 얇은 끝-끝 (뼈대 → 살)

```
[주제 고르기]      ← 창고에 이미 있음(주제 필터·트렌드)
   ↓
[재료 조립]        ← ★자체구축: 확인된 팩트 + (미확인이면 [검수:]) + 우리 글 예시 3편(가려진 본문) + 금지어 + 지킬 규칙
   ↓                 = 이게 곧 Claude API에 넘길 입력(프롬프트)
[API 초안 생성]    ← Claude API 호출 → 받은 초안 즉시 마스킹 → 파일로 저장
   ↓
[화면에서 열어 읽기] ← 뷰어에 새 읽기 화면
   ┈┈┈┈┈┈┈┈┈┈┈┈ (여기까지 1차 = 뼈대. 가장 위험한 '새로운 것'을 끝-끝으로 증명)
   ↓
[화면 편집기로 고쳐 저장] ← 2차. /fact/save 안전장치 패턴 재사용
   ↓
[완성 도장 + 손으로 홈페이지 발행] ← 3차. '완성' 도장 = 사람 승인(불변 8). 발행은 손(불변 7)
   ↓
[완성본이 다음 초안 예시로]   ← 3차. '학습'의 최소 형태(불변 3 위반 아님 — 내 문체 기록)
```

뼈대(1차)가 API 연동이라는 **가장 새롭고 위험한 조각**을 통째로 검증한다. 편집·발행·보관은 그 위에 붙이는 살이다. 뼈대가 돌기 전엔 편집기·보관을 만들지 않는다.

## 3. 완성본을 어디에 저장하나 (불변 4·9)

- **불변 4가 강제**: "본문은 파일에만, DB엔 경로만." 완성 원고도 본문이다(우리가 새로 쓴 글). → 초안 텍스트·편집본 텍스트는 **파일**(예: `corpus/manuscripts/` 또는 `drafts/`, 코퍼스 규칙 따라 git 제외)에 두고, DB엔 **경로 + 메타만**.
- **기존 `review_pairs` 테이블은 쓰지 않는다**: 그 테이블은 Phase1 "구조만"으로 만든 자리인데 `draft_ai`·`draft_human`이 **TEXT 본문 컬럼**이라 불변 4와 어긋난다. 불변 9(컬럼 삭제 금지)에 따라 **손대지 않고 그대로 두되(숨김)**, 새 테이블을 멱등 ensure로 추가한다.
- **새 테이블 `manuscripts`(가칭)** — 멱등 CREATE TABLE IF NOT EXISTS:
  - 주제(item_name/topic) · 근거 팩트 요약 · 사용 모델명 · 상태('초안'/'검토중'/'완성') · **draft_path**(AI 원본, 절대 덮지 않음) · **edited_path**(내가 고친 것) · 생성 시각 · 발행 시각·발행처 메모.
  - before/after는 두 경로(draft_path 원본 보존 + edited_path)로 확보 → 무거운 이력 테이블 없이 '학습' 재료가 됨(YAGNI).
- 다음 초안의 예시: 같은 주제의 `manuscripts` 중 **'완성' 상태 1~2편의 edited_path**를 읽어(이미 가려진 본문) 프롬프트에 붙인다.

## 4. 마스킹·API·안전장치 (불변 1·7·8, 선례 재사용)

- **마스킹(불변 1) 두 지점**: ①API가 준 초안을 **파일로 쓰기 직전** 별도 모듈에서 `masking.mask_text` 통과 ②화면 편집본을 **저장할 때** `/fact/save`와 똑같이 `masking.mask_text` 통과. 프롬프트에 넣는 우리 글 예시도 **이미 가려진 본문**(body_pub_ref_path/body_clean_path)만 쓴다.
- **API 실패·비용·재시도**: 별도 모듈이 처리 — 타임아웃, **재시도 1회 상한**(선례 bulk_extract 무한루프 가드 정신), 실패 시 파일·DB에 아무것도 안 쓰고 화면에 사람 말로 사유 표시. 호출 1건당 모델·토큰(대략 비용)을 로그로 남긴다. 비용 상한은 **모델 선택 + 최대 길이 + 예시 편수**로 통제(결정 Q1·Q2·Q5).
- **쓰기 주소는 최소로 늘린다**: 지금 쓰기는 `/fact/save` 하나. B방향에서 늘어나는 쓰기 주소는 **2개까지**로 본다 — ①`/draft/save`(편집본 저장) ②`/draft/stamp`(상태 도장='완성' 승인·되돌리기). 초안 생성 트리거는 아래 결정 Q6에 따라 명령 또는 `/draft/new`. **나머지 8개 화면은 계속 읽기 전용(POST 405 유지).** 새 쓰기 주소는 전부 `/fact/save`의 안전장치 9개(출처 Origin/Host 검증·busy_timeout 60s·입력검증·303 재이동·저장 시 마스킹·삭제 없음·저장 실패 시 입력 보존·확인 대화상자 0·되돌리기)를 그대로 따른다.
- **불변 8(승인 후 발행)**: 자동 발행이 없으므로 '완성' 도장이 곧 사람 승인 게이트. 도장 없이 발행처 메모를 채우지 못하게 한다(개념상 승인 큐).

## 8. 불변 준수 확인

- **불변 1**: 초안은 파일 쓰기 직전 + 편집본은 저장 시 `masking.mask_text` 통과. 프롬프트 속 우리 글 예시는 이미 가려진 본문만.
- **불변 3**: '학습'은 내가 고친 완성본을 예시로 쓰는 것뿐. 카페 조회수를 성과·학습 근거로 쓰지 않음.
- **불변 4**: 초안·편집본 텍스트는 파일에만, `manuscripts` DB엔 경로+메타만. 원문(posts.body_raw) 안 건드림. review_pairs(TEXT 본문)는 쓰지 않고 그대로 둠.
- **불변 6**: 팩트 검수 완료된 사회복지만 첫 원고. 미확인 팩트는 재료 조립에서 `[검수:]`로 표시, 수치 단정 금지.
- **불변 7·8**: 자동 발행 0. '완성' 도장 = 사람 승인, 발행은 손으로.
- **불변 9**: 새 테이블·칸은 멱등 ensure, 컬럼 삭제 없음. 새 쓰기 주소도 삭제 기능 없음.
- **디자인 규칙**: 편집 화면·초안 화면은 만든 뒤 브라우저로 실물 확인이 완료 조건. AI가 만든 값(초안)을 보여주는 화면은 근거(어느 팩트·어느 예시로 썼는지)로 갈 길을 같은 화면에 둔다.
