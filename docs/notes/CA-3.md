# CA-3 작업노트 — 수동 투입 파이프라인 (2026-07-07)

## 만든 것

**`src/intake_manual.py`** — `inbox/<슬러그>/`의 글 1건을 표준양식(서비스기획서 v9 부록 A)으로 정리:
1. **본문 3버전**(corpus/에 파일 저장, DB엔 경로만 — 불변 4):
   - `body_raw` 원문 그대로(절대 수정 안 함) / `body_clean` 정제 / `body_pub_ref` **개인정보 마스킹본**
2. **문단 분리 + 역할 1차 태깅**(위치+키워드 규칙, 사람 검수 전제 — 확신 낮으면 표시) + contains_fact/cta
3. **이미지 메타**(meta.json의 반자동 입력 → post_images): 출처·재사용범위·인물/로고/글자·문단위치
4. **DB 저장**: posts(upsert) + post_paragraphs + post_images + keywords/post_keywords + staff
5. **사람용 요약(A안)** 콘솔 출력 + `out/intake/<슬러그>_요약.md`

**`src/masking.py`** — 개인정보 마스킹 **단일 출처**(동기화 필수 쌍 준수). 패턴을 DB(rulebook_pii_patterns, CA-1 적재)에서 읽어 CA-3(정제)와 CA-4(린터)가 같은 것을 참조. 직원 실명은 staff 테이블(CA-2 적재)을 이름목록으로 재사용.

## 시연 결과 (기획서 부록 A 샘플 — 임상심리사2급)

`examples/intake_sample/임상심리사2급_응시자격/`(body.txt + meta.json, 합성 개인정보 3종 심어둠 — git 추적됨)로 검증. 실제 글은 `inbox/`(git 제외)에 넣는다. 데모 실행: `python src/intake_manual.py examples/intake_sample/임상심리사2급_응시자격`
- 문단 8개 역할: 1도입 2·3조건설명 4주의사항 5배경설명 6절차안내 7마무리 8CTA → 부록 A 구조와 거의 일치
- 이미지 6개: 인물 분위기사진·전화번호 상담배너 2개가 `image_pattern_only`(원본 재사용 금지)로 표시
- **마스킹 100%**: `010-1234-5678`→[전화번호], `가상인쌤`→[담당자], 오픈채팅 URL→[오픈채팅링크]. 원문(body_raw)엔 그대로 보존.

## 버그 잡은 것

- 오픈채팅 정규식이 `)으로`까지 과잉 포착 → 괄호·한글 앞에서 멈추도록 수정(load_rulebook 시드 + DB 재적재). 마스킹은 한 곳(DB)만 고치면 CA-3·CA-4 양쪽 반영.
- 문단1(인사말)이 '조건설명'으로 오태깅 → 첫 문단 도입 cue·마무리 cue를 키워드보다 우선 적용.
