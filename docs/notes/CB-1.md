# CB-1 작업노트 — 카드뉴스 조건 카드 시제품 (2026-07-07)

## 만든 것

**"AI는 내용 JSON만, 렌더링은 템플릿"** 방식(CLAUDE.md 현재결정 5) 구현.

- `templates/brand.css` — 브랜드 색·글꼴 **단일 정의**(디자인 규칙). 색은 CSS 변수로만, 개별 템플릿에 하드코딩 금지(테스트로 강제). ※ 현재 색상은 **시작 제안(플레이스홀더)** — 브랜드 확정 시 이 파일만 바꾸면 전 카드 반영.
- `templates/card_condition.html` — 조건 정리 카드 템플릿(인스타 세로 1080×1350).
- `src/cardnews.py` — 콘텐츠 JSON → 템플릿 치환 → **헤드리스 Chrome으로 PNG 캡처**. 새 파이썬 의존성 없음(이미 깔린 Chrome 사용). 행 값은 HTML 이스케이프(주입 방지), note만 `<b>` 허용.
- `examples/cardnews/*.json` — 룰북 실제 팩트로 만든 샘플 3장(사회복지사2급 / 기사 응시자격 / 학사학위 — 최상위 빈도 카테고리).
- `tests/test_cb1.py` — 9개 통과(치환 누락·이스케이프·디자인 규칙·팩트출처 표기·실렌더 스모크).
- 출력: `out/cardnews/*.png` (git 제외).

## 불변 준수

- **불변 6(팩트는 룰북에서만)**: 각 카드 값은 룰북 SECTION A 출처를 카드 하단에 명시("팩트 출처 · 통합룰북 V4.2"). 샘플 JSON도 `source` 필수(테스트 강제).
- **기간·비용은 단정 수치 금지**: 확정 팩트(학점·과목·실습)만 본문에, 기간·비용은 "개인 조건에 따라 달라져요" 안내문으로 처리.
- **불변 1·8(개인정보·CTA)**: 카드에 전화번호·오픈채팅·상담유도 CTA 없음. 발행 채널 유도는 사람이.

## 렌더링 방식 메모

- Playwright/Pillow 미설치 확인 → 대신 설치된 Chrome을 `--headless=new --screenshot --window-size=1080,1350`로 사용. 한글(맑은 고딕) 선명하게 렌더됨.
- brand.css를 `<link>` 상대경로로 부르므로 임시 HTML을 `templates/` 안에 생성 후 캡처·삭제.
